# Tool-Based Validation

## Objective
To validate cloud security controls using automated tools.

## Tool Used
Wiz

## What was validated
- Misconfigurations (e.g., public S3, open security groups)
- IAM risks (over-permissioned roles)
- Vulnerabilities in compute resources

## Attack Path Example
Public EC2 → IAM Role → S3 Access

## Outcome
- Risks identified and validated
- High-risk issues prioritized
- Supports audit findings with real data
