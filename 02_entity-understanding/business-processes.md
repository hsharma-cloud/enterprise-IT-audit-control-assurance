## 1. Purpose

The purpose of this document is to describe the key business processes supported by the systems within the audit scope. Understanding these processes allows auditors to identify how IT systems enable business operations and where risks may impact critical activities.

---

## 2. Overview

The organization relies on IT systems to support core business functions, including user access management, application usage, and operational workflows. These processes depend on secure, reliable, and controlled IT environments.

---

## 3. Key Business Processes

---

### 3.1 User Access Management

**Description:**
Process for granting, modifying, and revoking user access to systems and applications.

**Key Activities:**

* User onboarding and account creation
* Role assignment and access provisioning
* Access modification based on role changes
* User offboarding and access removal

**Systems Involved:**

* Active Directory
* Cloud Identity Provider
* Access Management Tools

---

### 3.2 Authentication and Authorization

**Description:**
Process of verifying user identity and granting appropriate system access.

**Key Activities:**

* User authentication (passwords, MFA)
* Authorization based on roles and permissions
* Session management and access validation

**Systems Involved:**

* Identity Providers
* Application Authentication Services

---

### 3.3 Privileged Access Management

**Description:**
Management of elevated access rights for administrative users.

**Key Activities:**

* Assignment of privileged roles
* Monitoring of privileged activities
* Periodic review of privileged accounts

**Systems Involved:**

* Privileged Access Management (PAM) tools
* Administrative interfaces

---

### 3.4 Security Monitoring and Incident Response

**Description:**
Detection and response to security events and incidents.

**Key Activities:**

* Log collection and analysis
* Alert generation and investigation
* Incident response and remediation

**Systems Involved:**

* SIEM platform
* Monitoring tools

---

### 3.5 Change and Configuration Management

**Description:**
Process for managing system changes and maintaining secure configurations.

**Key Activities:**

* Change request and approval
* Deployment of updates and patches
* Configuration baseline enforcement

**Systems Involved:**

* Change management systems
* Configuration tools

---

## 4. Process Flow Overview

Each business process follows a structured workflow:

1. Request or trigger (e.g., user onboarding, change request)
2. Approval and validation
3. Execution of action (e.g., access provisioning)
4. Logging and monitoring
5. Periodic review and validation

## 6. Risks in Business Processes

Key risks include:

* Unauthorized access due to improper provisioning
* Delayed removal of user access
* Excessive privileges assigned to users
* Lack of monitoring for critical activities
* Unauthorized or unapproved system changes

---

## 7. Control Points

Controls are implemented at critical points in each process:

* Access approval workflows
* Authentication mechanisms (e.g., MFA)
* Logging and monitoring controls
* Change approval processes

Business processes depend on:

* Logging and monitoring systems
